# ⛪ Parish Connect - Mobile Client

This repository contains the **Flutter mobile application client** for Parish Connect, an essential tool for managing and collecting accurate, organized data across Small Christian Communities (SCCs), Mission Stations, Deaneries, and Parishes.

## ✨ About the Application

The mobile client is the primary interface used by community and church leaders for:

* **Data Entry:** Inputting new records (e.g., membership, financials).
* **Offline Data Collection:** Storing data locally and syncing when connectivity is available (Planned).
* **Viewing Reports:** Accessing relevant statistics and summaries generated by the backend API.
* **User Authentication:** Secure login and session management.

## 💻 Tech Stack

* **Framework:** **Flutter** (version X.Y.Z)
* **Language:** **Dart**
* **State Management:** (**Riverpod**)
* **HTTP Client:** **`dio`** or **`http`**
* **Local Database:** (e.g., **`shared_preferences`** or **`sqflite`** for caching/offline data)

## 🚀 Getting Started

Follow these steps to get the Parish Connect mobile client running locally.

### Prerequisites

1.  **Flutter SDK:** Ensure you have Flutter version X.Y.Z or higher installed. [Installation Guide](https://flutter.dev/docs/get-started/install)
2.  **Code Editor:** VS Code or Android Studio with Flutter/Dart plugins.
3.  **Running Backend:** The **Node.js API server** must be running and accessible. Refer to the separate **`backend/README.md`** for setup instructions.

### Installation

1.  **Clone this repository:**
    ```bash
    git clone https://github.com/weslife911/PARISH_CONNECT.git
    cd futru_scc_app # Adjust folder name if needed
    ```

2.  **Fetch dependencies:**
    ```bash
    flutter pub get
    ```

3.  **Configure API Endpoint:**
    * Locate the configuration file, typically at `lib/core/config/api_config.dart`.
    * Set the **`BASE_API_URL`** to point to your running Node.js backend.

    > **Note:** When using the **Android Emulator**, you must use the IP address **`http://10.0.2.2:[PORT]`** to connect to `localhost` on your host machine.

4.  **Run the App:**
    ```bash
    flutter run
    ```
    Select a connected device or emulator (iOS, Android, Web, or Desktop).
